## 简介

TandOrbit 是一个 **Mac + Windows 双机双屏智能协同管理平台**。它通过软件统一管理两台计算机、两台显示器及外设，实现真正意义上的"无感切换"。

**核心理念：让两台电脑像一台电脑一样自然工作。**

### 主要功能

- 🖥️ **显示器管理** — 自动切换显示器输入源、启用/禁用显示器
- ⌨️ **键鼠共享** — 通过 Deskflow 实现跨平台鼠标键盘共享
- 🔌 **电源管理** — 自动唤醒/休眠远程计算机
- 🔊 **音频切换** — 自动切换音频输出设备
- 📋 **剪贴板同步** — 跨平台剪贴板内容同步
- ⚡ **一键切换** — 快捷键瞬间切换工作模式

## 安装

### 方式一：使用预编译安装包（推荐）

1. 前往 [GitHub Releases](https://github.com/signxer/TandOrbit/releases) 页面
2. 下载对应平台的安装包：
   - macOS: `TandOrbit-macOS.dmg`
   - Windows: `TandOrbit-Windows.zip`
3. 安装并运行

### 方式二：从源码运行

```bash
# 1. 克隆仓库
git clone https://github.com/signxer/TandOrbit.git
cd TandOrbit

# 2. 安装依赖
pip install -r requirements.txt

# 3. 运行 Mac 端
python -m app.main

# 4. 运行 Windows 端（在 Windows 电脑上）
python -m app.agent_main
```

### 方式三：开发模式安装

```bash
pip install -e ".[dev]"
```

## 配置

### 配置文件位置

配置文件位于：`~/.tandorbit/config.yaml`

首次运行时会自动创建默认配置文件。

### 配置示例

```yaml
# 显示器配置
display:
  primary_id: 1        # 主显示器 ID
  secondary_id: 2      # 副显示器 ID

# Windows Agent 配置
windows:
  host: "192.168.1.100"  # Windows 电脑的 IP 地址
  port: 5000              # Agent 服务端口

# Deskflow 配置
deskflow:
  auto_restart: true      # 自动重启
  server_host: "192.168.1.100"  # Deskflow 服务器地址
  server_port: 24800      # Deskflow 端口
  client_name: "mac"      # 客户端名称

# BetterDisplay 配置（macOS）
betterdisplay:
  cli_path: "/Applications/BetterDisplay.app/Contents/MacOS/betterdisplaycli"

# 音频配置
audio:
  mac_output: "AirPods"      # Mac 音频输出设备
  windows_output: "USB DAC"  # Windows 音频输出设备

# 快捷键配置
hotkeys:
  switch_mac: "Ctrl+Alt+1"
  switch_windows: "Ctrl+Alt+2"
  switch_share: "Ctrl+Alt+3"

# 日志配置
log_level: "INFO"
log_dir: "logs"
log_retention_days: 30
```

### 配置说明

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `display.primary_id` | 主显示器 ID | 1 |
| `display.secondary_id` | 副显示器 ID | 2 |
| `windows.host` | Windows Agent IP | 192.168.1.100 |
| `windows.port` | Agent 端口 | 5000 |
| `deskflow.server_host` | Deskflow 服务器 IP | 192.168.1.100 |
| `deskflow.server_port` | Deskflow 端口 | 24800 |
| `log_level` | 日志级别 | INFO |

## 使用方法

### 第一步：配置 Windows Agent 开机自启

**推荐：使用安装脚本（需管理员权限）**

1. 将 `TandOrbitAgent.exe` 和 `scripts/install_agent.bat` 放在同一目录
2. 右键 `install_agent.bat` → **以管理员身份运行**
3. 脚本会自动：
   - 创建 Windows 计划任务（开机自启）
   - 配置防火墙放行 5000 端口
   - 立即启动 Agent

**手动启动：**

```bash
python -m app.agent_main
```

或者：

```bash
TandOrbitAgent.exe
```

Agent 启动后会在 `0.0.0.0:5000` 监听 HTTP 请求。

### 第二步：配置 Wake on LAN（WoL）

如果 Mac 常开、Windows 会关机，需要配置 WoL 让 Mac 能自动唤醒 Windows：

**Windows 端设置：**

1. 进入 BIOS → 开启 Wake on LAN
2. 设备管理器 → 网络适配器 → 属性 → 电源管理 → 勾选"允许此设备唤醒计算机"
3. 设备管理器 → 网络适配器 → 属性 → 高级 → 开启 Wake on Magic Packet

**Mac 端配置：**

在 `~/.tandorbit/config.yaml` 中添加 Windows 的 MAC 地址：

```yaml
wol:
  mac_address: "AA:BB:CC:DD:EE:FF"  # Windows 网卡 MAC 地址
  broadcast_ip: "192.168.1.255"     # 局域网广播地址
```

> 查看 Windows MAC 地址：在 Windows 上运行 `ipconfig /all`，找到物理地址。

配置完成后，切换到 Windows/共享模式时，TandOrbit 会自动：
1. 发送 WoL 魔术包唤醒 Windows
2. 等待 Windows Agent 上线（最多 60 秒）
3. 执行后续切换操作

### 第二步：启动 Mac 端

在 Mac 上运行：

```bash
python -m app.main
```

或者使用预编译版本，双击 `TandOrbit.app`。

### 第三步：切换工作模式

通过以下方式切换模式：

1. **主界面按钮** — 点击界面上的模式按钮
2. **系统托盘** — 右键托盘图标选择模式
3. **快捷键** — 使用 `Ctrl+Alt+1/2/3`

## 工作模式详解

### Mac 模式

```
显示器 1 → Mac
显示器 2 → Mac
Windows → 待机
```

- 两台显示器都连接到 Mac
- Windows 自动进入待机状态
- 适合纯 Mac 工作场景

### Windows 模式

```
显示器 1 → Windows
显示器 2 → Windows
Mac → 休眠
```

- 两台显示器都连接到 Windows
- Mac 自动休眠
- 适合纯 Windows 工作场景

### 共享模式（推荐）

```
显示器 1 → Mac
显示器 2 → Windows
键鼠共享 → Deskflow
```

- 一台显示器给 Mac，一台给 Windows
- 通过 Deskflow 实现键鼠共享
- 鼠标可以自由在两个屏幕之间移动
- **这是 TandOrbit 最具特色的模式**

## 常见问题

### Q: 两台电脑如何连接？

A: 两台电脑需要在同一局域网内。TandOrbit 通过 HTTP API 通信，Mac 端是客户端，Windows 端是服务端（Agent）。

### Q: 如何查看 Windows 的 IP 地址？

A: 在 Windows 上打开命令提示符，输入 `ipconfig`，找到 IPv4 地址。

### Q: BetterDisplay 是什么？

A: BetterDisplay 是 macOS 上的显示器管理工具，TandOrbit 通过它的 CLI 接口控制显示器。官网：https://github.com/waydabber/BetterDisplay

### Q: Deskflow 是什么？

A: Deskflow 是开源的键鼠共享软件（原 Synergy/Barrier），允许一套键鼠控制多台电脑。官网：https://github.com/deskflow/deskflow

### Q: 支持多台显示器吗？

A: 目前支持 2 台显示器（每台电脑各一台）。未来版本会支持更多显示器配置。

### Q: 支持 Linux 吗？

A: 目前仅支持 macOS + Windows。Linux 支持在规划中。

## 故障排除

### 问题：无法连接到 Windows Agent

**检查清单：**

1. Windows Agent 是否已启动？
2. Windows 防火墙是否放行 5000 端口？
3. 两台电脑是否在同一局域网？
4. 配置文件中的 IP 地址是否正确？

**测试连接：**

```bash
# 在 Mac 上测试
curl http://192.168.1.100:5000/api/health
```

### 问题：显示器切换不工作

**检查清单：**

1. BetterDisplay 是否已安装？（macOS）
2. MultiMonitorTool 是否在 PATH 中？（Windows）
3. 显示器 ID 是否正确？

### 问题：Deskflow 无法连接

**检查清单：**

1. Deskflow 是否已安装？
2. Deskflow 服务器是否在 Windows 上运行？
3. 端口 24800 是否被占用？

**重启 Deskflow：**

```bash
# macOS
osascript -e 'quit app "Deskflow"'
open -a Deskflow

# Windows
taskkill /IM Deskflow.exe /F
start Deskflow.exe
```

### 问题：日志在哪里？

日志文件位于：
- macOS: `~/TandOrbit/logs/`
- Windows: `C:\Users\你的用户名\TandOrbit\logs\`
