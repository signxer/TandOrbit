# TandOrbit 软件设计说明书（Software Design Specification）

**项目名称：** TandOrbit

**版本：** V1.0

**文档版本：** Draft 1

**语言：** Python 3.13

**GUI：** PySide6

---

# 1. 项目目标

TandOrbit 是一个针对 **macOS + Windows 双机双屏** 场景开发的桌面协同软件。

目标：

* 统一管理两台计算机
* 自动管理显示器状态
* 自动管理 Deskflow
* 自动恢复显示器布局
* 自动执行跨平台切换
* 对外提供统一 API

整个系统采用 **State Machine + Plugin Architecture**。

所有动作均由状态驱动，而不是脚本驱动。

# 3. 模块划分

## 3.1 GUI

负责：

* 主界面
* 设置
* 日志
* 托盘
* 快捷键

禁止直接调用系统命令。

所有操作均发送至：

```
Controller
```

---

## 3.2 Controller

唯一入口。

例如：

```python
controller.switch_mode(
    Mode.SHARE
)
```

Controller：

负责：

* 参数检查
* 状态检查
* 调度 Scheduler

不负责业务逻辑。

---

## 3.3 Scheduler

负责：

动作排序。

例如：

Share Mode：

```
关闭Mac副屏

↓

Windows Duplicate

↓

Deskflow Restart

↓

验证

↓

结束
```

Scheduler 必须保证：

动作失败时：

```
Rollback
```

恢复。

---

## 3.4 EventBus

整个系统采用事件驱动。

例如：

```
MonitorDisabled

↓

EventBus

↓

Deskflow Restart
```

再例如：

```
Windows Online

↓

Wake Success

↓

恢复布局
```

任何模块不得互相调用。

只能：

```
Publish

Subscribe
```

# 16. 开发规范

采用以下原则：

* **SOLID**：所有模块遵循单一职责、依赖倒置等原则，降低耦合。
* **依赖注入（DI）**：Controller 依赖抽象接口，不直接依赖具体插件实现，便于测试和扩展。
* **插件注册机制**：通过注册表加载插件，避免硬编码。
* **命令模式（Command Pattern）**：所有切换动作封装为可执行命令，支持队列、重试与回滚。
* **事件总线（Event Bus）**：模块间通过事件通信，避免直接相互调用。
* **有限状态机（FSM）**：所有工作模式切换均由状态机驱动，确保状态一致性。
* **接口优先**：所有平台能力必须先定义抽象接口，再实现具体 Provider。
* **单元测试优先**：Controller、StateManager、Scheduler、各 Plugin 均应具备可独立测试能力。
